# AI Equation to DOCX Generator (Web App)

This project provides a web-based tool to convert text (with equations in natural language) into a DOCX file. It uses the Gemini AI to generate Python code that creates the document, and then executes this code on the server to produce the DOCX file. The user can then download the generated file and view the AI-generated Python code.

## Features
- Enter text with embedded equations in natural language via a web interface.
- Uses Gemini AI to generate Python code (`python-docx`, `math2docx`) for DOCX creation.
- Executes the generated Python code on the server.
- Allows download of the generated DOCX file.
- Displays the raw AI-generated Python code in the web interface.
- Simple local web server using Flask.

## Requirements
- Python 3.8+
- Dependencies listed in `requirements.txt`:
  - `google-generativeai`
  - `python-docx`
  - `math2docx`
  - `python-dotenv`
  - `Flask`

## Setup
1.  **Clone the repository:**
    ```bash
    git clone <repository_url>
    cd <repository_directory>
    ```
2.  **Create a virtual environment (recommended):**
    ```bash
    python -m venv venv
    source venv/bin/activate  # On Windows: venv\Scripts\activate
    ```
3.  **Install dependencies:**
    ```bash
    pip install -r requirements.txt
    ```
4.  **Set your Gemini API key:**
    - Get your API key from Google AI Studio (formerly MakerSuite).
    - Create a file named `.env` in the project root directory.
    - Add your API key to the `.env` file like this:
      ```
      GEMINI_API_KEY="your_api_key_here"
      ```
    - Alternatively, you can set it as an environment variable directly in your system.

## Usage
1.  **Run the Flask web application:**
    ```bash
    python app.py
    ```
2.  Open your web browser and go to `http://127.0.0.1:5001` (or the address shown in the terminal).
3.  Enter your text (with equations) in the input area.
4.  Click **"Generate DOCX & Show AI Code"**.
5.  The AI-generated Python code will be displayed.
6.  If successful, a download link for `output.docx` will appear. Click it to download the file.
7.  Any errors during the process will be displayed on the page.
8.  Generated DOCX files are stored in the `generated_files` directory on the server.

## File Structure
- `app.py`: Main Flask application file. Handles web routes, AI interaction, code execution, and file serving.
- `ai_agent.py`: Contains the logic to interact with the Gemini API and generate Python code.
- `index.html`: The main HTML page for the web interface (served from the root, but could be moved to a `templates` folder if the app grows).
- `requirements.txt`: Lists Python package dependencies.
- `.env` (you create this): Stores your Gemini API key.
- `generated_files/`: Directory where generated DOCX files are stored (created automatically).
- `README.md`: This file.
- `.gitignore`: Specifies intentionally untracked files that Git should ignore.
- `docx_writer.py`: (Currently unused by the web app but contains helper functions for direct DOCX creation).
- `main.py`: (Legacy Tkinter application, not used by the web app).


## Important Security Note
This application uses `exec()` to run Python code generated by an AI. While the prompt to the AI is designed to produce benign `python-docx` scripts, executing arbitrary code always carries risks. This setup is intended for local development and experimentation. Be cautious if exposing this application to untrusted users or running it in a sensitive environment.
